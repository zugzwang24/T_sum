# 황금을 찾아라 데이터 기능 고도화 정리

이 문서는 카페 상권 추천 서비스의 데이터 전처리, 추천 점수 계산, 이상치 보정, 데이터 신뢰도 표시, 로컬 LLM 해설 구조를 보고서 작성용으로 정리한 것이다.

## 1. 고도화 배경

초기 서비스는 서울 공공데이터 기반으로 카페 창업 후보 상권을 추천하는 구조였다. 주요 지표는 객단가, 2030 매출비율, 2030 유동인구, 카페전환효율, 시간대별 매출비중이었다.

하지만 `카페전환효율 = 당월 카페 매출건수 / 월 유동인구추정` 방식은 일부 행정동에서 값이 과도하게 튀는 문제가 있었다. 유동인구 추정치가 작거나 특정 상권의 매출건수가 집중된 경우, 실제보다 전환효율이 매우 높게 계산되어 추천 순위가 왜곡될 수 있었다.

따라서 이번 고도화의 목적은 단순히 높은 점수를 보여주는 것이 아니라, 추천 결과의 해석 가능성과 데이터 신뢰도를 함께 제공하는 것이다.

## 2. 데이터 전처리 고도화

기존에는 2030 중심의 파생변수에 가까웠지만, 현재는 추천 점수를 CSV에 고정 저장하지 않고 백엔드에서 조건별로 동적 계산하도록 변경했다.

전처리 결과 파일:

```text
data_analysis/data/processed/cafe_area_features.csv
```

주요 feature:

```text
행정동_코드
행정동_코드_명
업종명
당월_매출_금액
당월_매출_건수
총매출건수
객단가
카페전환효율
2030_매출비율
2030_유동인구비율
월_유동인구추정
연령대별 매출건수
연령대별 매출금액
연령대별 유동인구
시간대별 매출비중
시간대별 유동인구비중
```

시간대는 다음 6개로 통일했다.

```text
새벽: 00~06
오전: 06~11
점심: 11~14
오후: 14~17
저녁: 17~21
심야: 21~24
```

## 3. 타깃 연령대 일반화

기존에는 2030 기준이 강했지만, 현재는 사용자가 타깃 연령대를 선택할 수 있도록 변경했다.

선택 가능 연령대:

```text
10대, 20대, 30대, 40대, 50대, 60대 이상
```

예를 들어 `targetAges=20,30`이면 20대와 30대를 합산하고, `targetAges=40,50`이면 40대와 50대를 합산하여 추천 점수를 계산한다.

타깃 매출비율:

```text
타깃 매출비율 = 선택 연령대 매출건수 합계 / 총매출건수
```

타깃 유동인구비율:

```text
타깃 유동인구비율 = 선택 연령대 유동인구 합계 / 총 유동인구
```

타깃 전환효율:

```text
타깃 전환효율 = 선택 연령대 매출건수 합계 / (선택 연령대 유동인구 합계 × 30)
```

## 4. 추천 점수 계산 방식

추천 점수는 CSV에 저장하지 않고, 백엔드에서 사용자의 조건에 따라 계산한다.

현재 기본 점수 구성:

```text
카페전환효율: 35%
타깃 매출비율: 30%
선택 시간대 매출비중: 25%
객단가: 10%
```

각 지표는 행정동 전체 분포에서 min-max normalization을 적용한 뒤 가중합으로 계산한다.

기본점수:

```text
기본점수 =
  정규화된 카페전환효율 × 0.35
+ 정규화된 타깃 매출비율 × 0.30
+ 정규화된 선택 시간대 매출비중 × 0.25
+ 정규화된 객단가 × 0.10
```

최종적으로 100점 만점으로 환산한다.

## 5. 이상치 보정 점수

카페전환효율처럼 일부 상권에서 값이 과도하게 튀는 지표를 완화하기 위해 `useAdjustedScore=true` 옵션을 추가했다.

일반 점수는 전체 최솟값과 최댓값을 기준으로 정규화한다. 반면 이상치 보정 점수는 상위/하위 극단값의 영향을 줄이기 위해 1%~99% percentile 범위를 기준으로 정규화한다.

API 예시:

```text
GET /api/recommend?time=저녁&targetAges=20,30&limit=10&useAdjustedScore=true
```

프론트엔드에서는 `이상치 보정 점수 사용` 토글을 추가했고, 기본값은 보정 점수를 사용하는 방향으로 설정했다.

## 6. 데이터 신뢰도 지표 추가

추천 점수와 별개로 `데이터 신뢰도`를 추가했다. 이는 점수가 높더라도 데이터 해석에 주의가 필요한 상권을 구분하기 위한 보조 지표다.

신뢰도 계산에 사용한 요소:

```text
총 매출건수 규모
총 유동인구 규모
선택한 타깃 연령대 유동인구 규모
카페전환효율 이상치 여부
```

신뢰도 등급:

```text
80점 이상: 높음
60점 이상: 보통
40점 이상: 주의
40점 미만: 낮음
```

신뢰도는 추천 점수에 완만하게 반영된다.

```text
신뢰도 보정계수 = 0.55 + (신뢰도 점수 / 100) × 0.45
최종점수 = 기본점수 × 신뢰도 보정계수
```

초기에는 신뢰도 감점 폭을 최대 20% 수준으로 두었지만, 신뢰도 40점 수준의 상권도 추천 1위에 올라오는 문제가 있었다. 따라서 감점 폭을 더 크게 조정하여 신뢰도 40점인 경우 약 73% 수준의 보정계수를 적용하도록 변경했다.

또한 최종 TOP 10 정렬 정책을 단순 점수순에서 `신뢰도 60점 이상 우선 정렬`로 변경했다. 즉 신뢰도가 60점 미만인 상권은 점수가 높더라도 기본 추천 상단에 노출하지 않고 `검토 후보`로 분리한다.

```text
신뢰도 60점 이상: 안정 추천
신뢰도 60점 미만: 검토 후보
```

이 방식은 데이터가 불안정한 상권을 완전히 숨기지는 않지만, 사용자가 이를 1순위 추천으로 오해하지 않도록 한다.

## 7. 주의사항 자동 생성

상권 상세 화면에는 `주의할 점`을 추가했다.

주의사항 예시:

```text
매출건수 표본이 작은 편이라 전환효율이 실제보다 크게 흔들릴 수 있습니다.
총 유동인구가 낮은 편이라 특정 매장의 영향이 크게 반영됐을 수 있습니다.
선택한 타깃 연령대 유동인구 표본이 작아 타깃 적합도 해석에 주의가 필요합니다.
카페전환효율이 상위 1% 수준으로 높아 이상치 가능성이 있습니다.
카페전환효율이 하위 1% 수준으로 낮아 데이터 누락 또는 특수 상권 여부를 확인해야 합니다.
선택 시간대 매출 집중도가 유동인구보다 높게 나타나 재방문/목적형 소비 영향일 수 있습니다.
```

이를 통해 사용자는 추천 점수뿐 아니라 점수가 높게 나온 이유와 조심해야 할 해석 포인트를 함께 확인할 수 있다.

## 8. 프론트엔드 반영 내용

프론트엔드에는 다음 기능을 추가했다.

```text
타깃 연령대 선택 UI
이상치 보정 점수 사용 토글
추천 카드의 데이터 신뢰도 배지
상권 상세 패널의 기본점수/최종점수 표시
상권 상세 패널의 주의사항 표시
상권 비교 패널의 신뢰도 표시
로컬 LLM 해설 사용 여부 표시
```

화면에서 사용자는 단순히 TOP 10 순위를 보는 것이 아니라, 각 상권이 왜 추천되었는지, 그리고 데이터 해석에서 어떤 부분을 조심해야 하는지 확인할 수 있다.

## 9. 로컬 LLM 해설 구조

로컬 LLM은 추천 순위나 점수를 계산하지 않는다. 순위와 점수는 백엔드의 정량 지표로만 계산한다.

LLM의 역할:

```text
추천 결과 해설
지표 해석
운영 전략 문장화
주의사항 설명
```

현재 Ollama 기반 로컬 LLM을 사용할 수 있도록 구성되어 있으며, 기본 모델은 `llama3.1`이다.

Ollama 연결이 실패하면 규칙 기반 설명으로 fallback한다. 따라서 LLM이 없어도 추천 서비스 자체는 동작한다.

## 10. 주요 API

추천 API:

```text
GET /api/recommend?time=저녁&targetAges=20,30&industry=커피-음료&limit=10&useAdjustedScore=true
```

상권 상세 API:

```text
GET /api/areas/11440660?time=저녁&targetAges=20,30&useAdjustedScore=true&ai=true
```

상권 비교 API:

```text
GET /api/compare?areaA=11440660&areaB=11440710&time=저녁&targetAges=20,30&useAdjustedScore=true
```

API 응답에 추가된 주요 필드:

```text
score: 최종 추천점수
baseScore: 신뢰도 보정 전 기본점수
reliabilityFactor: 신뢰도 보정계수
dataQuality: 데이터 신뢰도 정보
cautions: 해석상 주의사항
metrics: 상권 주요 지표
scoreBreakdown: 점수 구성 요소
reasons: 규칙 기반 추천 사유
aiReason: 로컬 LLM 또는 규칙 기반 해설
```

## 11. 검증 결과 요약

검증 스크립트:

```text
analysis/validate_recommendation.js
```

검증 내용:

```text
시간대별 TOP 10 추천 결과 비교
2030 타깃과 4050 타깃 추천 결과 비교
일반 점수와 이상치 보정 점수의 TOP 10 민감도 비교
```

검증 결과, 이상치 보정 점수를 사용하면 카페전환효율이 과도하게 높은 일부 행정동의 영향이 줄어들고, 서교동, 신촌동, 연남동처럼 표본 규모가 크고 타깃 매출비율이 안정적인 상권이 더 상위로 올라오는 경향을 확인했다.

예를 들어 전환효율이 매우 높게 나온 둔촌1동은 일반 점수에서는 강하게 상위권에 나타났지만, 보정 점수에서는 신뢰도와 이상치 가능성이 반영되어 순위 영향이 완화되었다.

## 12. 보고서 작성 시 해석 방향

이 서비스의 추천 결과는 특정 상권이 무조건 좋다는 의미가 아니다. 추천 점수는 공공데이터 기반의 후보 탐색 지표이며, 실제 창업 의사결정에는 임대료, 경쟁 카페 수, 역세권 여부, 프랜차이즈 밀도, 상권 현장조사 등이 추가로 필요하다.

보고서에서는 다음 관점으로 설명하는 것이 적절하다.

```text
추천 점수는 후보 상권을 빠르게 선별하기 위한 1차 필터다.
카페전환효율은 유용하지만 이상치에 취약하므로 보정과 신뢰도 표시가 필요하다.
타깃 연령대와 운영 시간대에 따라 유망 상권은 달라질 수 있다.
LLM은 의사결정을 대신하지 않고, 정량 지표를 사용자가 이해하기 쉽게 해설하는 역할이다.
```
